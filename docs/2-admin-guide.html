ADMINISTRATOR GUIDE

Running exams on Matra

For examiners and invigilators · exams.matramedia.co.in/admin

1. Signing in & roles

Open /admin and sign in with the Google account your administrator authorised. Access is decided by your role, not a password the platform stores.

RoleCan do
OWNEREverything, including managing the team, organisations, and deleting data.
EXAMINERCreate & run exams, manage the question bank and papers, issue certificates, view results.
INVIGILATORMonitor exams live and assist students on exam day.
VIEWERRead-only access to results and analytics.

If your account is scoped to an institution, you see only that institution's exams, students, and question bank.

2. The exam lifecycle at a glance

  1. Create the batch (the exam) with its schedule and rules.
  2. Add questions — compose from the bank or upload a CSV.
  3. Upload the roster (roll number, name, email).
  4. Mark it Scheduled; share the exam code with students.
  5. Start it (Start Now) at exam time.
  6. Monitor live in Mission Control.
  7. End it, then view Results, Item Analysis, and issue Certificates.

3. Create a batch

From the dashboard, Create New Batch. Set the name, date/time (IST), duration, and (optionally) questions-per-student, passing percentage, and maximum attempts. Each batch is automatically assigned an 8-character exam code.

  • Institution — if more than one exists, pick the one this exam belongs to.
  • Show results to students — when off, students see "Exam submitted" but not a score. Toggleable any time.
  • Passing percentage / Max attempts — multiple attempts require a pass mark; each retry uses a different question selection.
  • Publicly listed — leave OFF for normal exams (reachable only by code). Turn ON only for open/practice events.
After creating, open the batch (Edit) to see its exam code and to download a batch summary — a one-page sheet with the code, schedule, counts, and a warning if no roster is set.

4. Add questions

Option A — Compose from the question bank

On the batch's Questions screen choose Compose paper. Add rules by topic, difficulty, and count (e.g. 10 × Vedanta easy). Live availability is shown per rule. The system copies approved bank questions into this paper, preferring least-recently-used items.

Option B — Upload CSV

Columns: question, option_a, option_b, option_c, option_d, correct (correct = A/B/C/D). UTF-8; Devanagari supported.

5. The question bank & review

The Question Bank tab holds reusable questions with metadata (topic, difficulty, Bloom's level, language, tags). Workflow: draft → in review → approved → retired.

  • Only approved questions can be composed into a paper.
  • Four-eyes rule: you cannot approve a question you authored — a different admin must approve it.
  • Editing the text or answer of an approved question sends it back to draft (new version) for re-review.
  • Questions are private to your institution; questions marked shared are visible to all.

6. Upload the roster

On the batch's Roster screen, provide roll number, name, and email for each student. Identity at entry is verified on roll number + email together — both must match. Without a roster, anyone with the code can enter under any name (low-stakes only; you'll see a warning).

7. Share the exam code & go live

  1. Mark the batch Scheduled.
  2. Share the 8-character exam code (notice board, admit card, or instructor). Students enter it at exams.matramedia.co.in/exam.
  3. At exam time, open the batch and choose Start Now. Students in the waiting room advance automatically.
Ask students to run the System Check at /check a day before, on the device they'll use.

8. Mission Control (exam day)

On an active batch, choose Live. The exam code is shown at the top with a copy button. The table refreshes automatically and shows each student's state:

  • In exam — active, with a recent heartbeat and a live answer count.
  • Disconnected — no heartbeat for 90 seconds. The student can simply reopen the exam link and resume; their answers are already saved.
  • Submitted — finished.
  • Flagged — elevated tab-switch / integrity signals (for review, not automatic action).

Grant extra time: use Extend time on a student's row. It takes effect within ~30 seconds and must be granted before their timer expires.

9. Results, analysis & certificates

  • Results — per-student scores; sortable; CSV export.
  • Item analysis & integrity report — difficulty/discrimination per question (flagged where weak), distractor spread, and the anomaly report. Includes a one-click Report Pack (.xlsx) — the hand-off artifact for the institution.
  • Certificates — Issue creates one certificate per eligible student (passing students when a pass mark is set). Print produces A4 certificates with a QR pointing to the public verification page. Students/employers verify at /verify.

10. Exam series (modular assessment)

Use the Series tab for a single mark-out-of-100 delivered as weighted modules across a term (e.g. 20-20-20-40).

  1. Create the series with module weights and an aggregate pass mark.
  2. Create a normal batch for each module and attach it to its slot (a make-up sitting can be attached too).
  3. Paste the series roster once — it syncs to every module's batch.
  4. Load results any time for the module-by-module breakdown and weighted aggregate; export CSV.

Module failures stay on the record even when the aggregate passes. Absence counts zero until a make-up is taken. Students see their running total after each module.

11. Team & organisations (owners)

The Team tab manages admin accounts and institutions. Add an admin by email + role (+ institution to scope them). They sign in with Google immediately — no invitation step. Create organisations here; batches can then be assigned to them.

12. Exam-day quick checklist

  • ☐ Roster uploaded and verified count looks right (batch summary).
  • ☐ Questions in the paper ≥ questions-per-student.
  • ☐ Exam code shared with students.
  • ☐ Students reminded to run /check beforehand.
  • ☐ At start time: Start Now → watch Mission Control.
  • ☐ After: End exam → Results → Report Pack → Certificates.
